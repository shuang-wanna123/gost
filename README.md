# GOST 多隧道管理脚本

一个基于 GOST v3 的多隧道管理脚本。适合快速部署 Shadowsocks 落地端和中转端。脚本带交互菜单，支持 systemd 守护、隧道列表、单独管理、批量操作、完整卸载，并在创建中转隧道后自动生成可直接导入 v2rayN 的 `ss://` 链接。

## 功能说明

脚本支持两种隧道模式。

落地隧道用于直接提供 Shadowsocks 服务端。可自定义监听端口、加密方式和密码。密码和端口都支持直接回车使用默认值。

中转隧道用于把本机端口转发到远端落地机。适合前后端分离部署。创建成功后会自动显示一个标准 `ss://` 导入链接，方便在 v2rayN 中一键导入使用。

脚本自带 `gg` 快捷命令。首次运行后会自动安装到 `/usr/local/bin/gg`，同时建立 `/usr/bin/gg` 软链接，后续可直接输入 `gg` 打开管理菜单。

## 支持环境

推荐系统为 Debian 12 或 Debian 13。其他使用 systemd 的常见 Linux 发行版通常也可运行。

脚本依赖 `bash`、`systemctl`、`tar`，下载 GOST 时需要系统里有 `curl` 或 `wget` 其中之一。

## 安装和运行

先把脚本保存为 `gg.sh`，再赋予执行权限并运行：

```bash
chmod +x gg.sh
./gg.sh
```

脚本启动后会自动初始化目录并安装快捷命令。以后直接执行下面这条即可进入菜单：

```bash
gg
```

如果当前 shell 里 `gg` 还没立即生效，可以重新登录一次终端，或者执行：

```bash
hash -r
```

## 菜单功能

主菜单包含添加隧道、隧道列表、隧道管理、批量操作和完全卸载。

添加隧道分为落地隧道和中转隧道。

隧道列表会显示名称、类型、运行状态、本地地址和目标地址。

隧道管理支持启动、停止、重启、查看日志、查看配置和删除单个隧道。

批量操作支持一键启动全部、一键停止全部和一键重启全部。

完全卸载会删除所有隧道配置、systemd 服务、GOST 程序和 `gg` 快捷命令。

## 默认配置

脚本里的默认值如下。

默认落地端口为 `8443`。

默认中转本地监听端口为 `51520`。

默认密码为 `Qwert1470`。

默认加密方式为 `chacha20-ietf-poly1305`。

这些参数都可以在创建时手动输入，也可以直接回车使用默认值。

## 落地端使用说明

创建落地隧道时，脚本会提示输入隧道名称、监听端口、加密方式和密码。

创建成功后，脚本会自动生成对应的 systemd 服务，并立即启动。

落地隧道本质上是一个 Shadowsocks 服务端。客户端连接时，需要填写落地端的公网 IP、端口、加密方式和密码。

## 中转端使用说明

创建中转隧道时，脚本会提示输入隧道名称、落地鸡 IP、落地鸡端口、本地监听端口、客户端加密方式和客户端密码。

这里的客户端加密方式和客户端密码，必须和落地端实际使用的值一致。因为中转只是转发，不负责改写 Shadowsocks 协议本身。

中转创建成功后，脚本会自动获取当前服务器公网 IP，并生成一条标准 `ss://` 链接。把这条链接直接导入 v2rayN 就能使用。

一个典型流程是这样的。

先在落地机创建一个落地隧道，比如监听 `8443`，密码 `Qwert1470`，加密方式 `chacha20-ietf-poly1305`。

再在中转机创建一个中转隧道，把本地端口比如 `58400` 转发到落地机的 `8443`，同时把客户端密码和加密方式填写成和落地端一致。

创建完成后，脚本会显示类似下面这样的导入链接：

```text
ss://Y2hhY2hhMjAtaWV0Zi1wb2x5MTMwNTpRd2VydDE0NzA@your_server_ip:58400#relay58400
```

把它复制到 v2rayN 里导入即可。

## 配置文件位置

GOST 程序默认安装到：

```text
/usr/local/bin/gost
```

隧道配置文件默认保存在：

```text
/etc/gost/tunnels
```

systemd 服务文件默认保存在：

```text
/etc/systemd/system/
```

快捷命令默认路径为：

```text
/usr/local/bin/gg
```

软链接路径为：

```text
/usr/bin/gg
```

## 常见问题

如果输入 `gg` 没反应，先确认文件是否存在：

```bash
ls -l /usr/local/bin/gg /usr/bin/gg
```

再确认脚本有执行权限：

```bash
chmod +x /usr/local/bin/gg
```

然后刷新 shell 命令缓存：

```bash
hash -r
```

如果仍然不行，重新登录终端再试。

如果创建隧道时报端口占用，脚本会尝试自动清理对应端口上的旧进程。若系统里没有 `fuser`，自动清理可能失效。

如果中转创建成功但导入后无法连接，先检查中转填写的加密方式和密码是否和落地端完全一致，再检查落地端端口是否可达。

## 查看日志

可通过脚本菜单进入单个隧道的日志查看。

也可以直接使用 `journalctl`：

```bash
journalctl -u gost-tun-隧道名 -f --no-pager
```

比如：

```bash
journalctl -u gost-tun-relay58400 -f --no-pager
```

卸载会删除全部隧道、全部 systemd 服务、GOST 程序和 `gg` 快捷命令。
